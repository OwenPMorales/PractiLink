    1. INTRODUCCIÓN

    La tesina profesional representa un trabajo de investigación y desarrollo que permite al estudiante demostrar las competencias adquiridas durante su formación académica. Este documento presenta el desarrollo de una aplicación multiplataforma utilizando tecnologías modernas de desarrollo de software.

    El objetivo principal de este proyecto es crear una solución tecnológica que resuelva una problemática real en el ámbito educativo, específicamente en la gestión de prácticas profesionales de estudiantes de ingeniería.

    2. MARCO TEÓRICO

    El marco teórico se fundamenta en los principios de desarrollo de software moderno, incluyendo:

    • Arquitecturas de software escalables
    • Desarrollo ágil y metodologías iterativas
    • Tecnologías web modernas (HTML5, CSS3, JavaScript)
    • Frameworks de desarrollo multiplataforma

    3. METODOLOGÍA

    La metodología utilizada sigue un enfoque de desarrollo ágil, dividido en las siguientes fases:

    1. Análisis de requisitos
    2. Diseño de la arquitectura
    3. Implementación del prototipo
    4. Pruebas y validación
    5. Despliegue y documentación

    4. DESARROLLO

    El desarrollo se realizó utilizando las siguientes tecnologías:

    • Frontend: HTML5, CSS3, JavaScript vanilla
    • Backend: Node.js con Express
    • Base de datos: MongoDB
    • Control de versiones: Git

    5. RESULTADOS

    Los resultados obtenidos demuestran el cumplimiento de los objetivos planteados:

    • Interfaz de usuario intuitiva y responsive
    • Funcionalidades completas de gestión
    • Integración exitosa con base de datos
    • Documentación técnica completa

    6. CONCLUSIONES

    El proyecto desarrollado cumple con los estándares de calidad requeridos para una tesina profesional. La aplicación resultante proporciona una solución efectiva para la gestión de prácticas profesionales, facilitando el proceso administrativo y mejorando la experiencia tanto para estudiantes como para tutores.

    7. BIBLIOGRAFÍA

    1. Pressman, R. S. (2014). Ingeniería del Software. Un enfoque práctico.
    2. Sommerville, I. (2011). Ingeniería del Software.
    3. Fowler, M. (2003). UML Distilled.
